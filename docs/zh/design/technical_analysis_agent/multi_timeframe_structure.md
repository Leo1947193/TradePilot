# ① 多周期结构分析

## 1. 模块职责与边界

### 范围内

| 职责          | 说明                                             |
| ------------- | ------------------------------------------------ |
| 均线计算      | 日线 SMA(20/50/200)、EMA(10/21)；周线 SMA(10/40) |
| 均线排列判定  | 根据价格与均线的相对位置，输出四级排列状态       |
| 日线趋势判定  | 综合均线斜率、价格位置、交叉信号，输出日线方向   |
| 周线趋势判定  | 综合周线均线与价格关系，输出周线方向             |
| 支撑/阻力识别 | 基于 52 周波段高低点，识别关键价格位并聚类合并   |
| 日线/周线共振 | 方向一致时增强信号，冲突时降级                   |

### 范围外

| 不做                               | 原因                        |
| ---------------------------------- | --------------------------- |
| 动量指标（RSI/MACD/ADX）           | 由 ② 动量与强度量化模块负责 |
| 成交量分析（OBV/突破确认）         | 由 ③ 价量关系分析模块负责   |
| 形态识别                           | 由 ④ 形态识别模块负责       |
| 风险指标（ATR/Beta/布林带）        | 由 ⑤ 风险指标计算模块负责   |
| 日内/tick 级别分析                 | 超出系统范围                |
| 信号评分与 `technical_signal` 聚合 | 由聚合器负责                |

## 2. 输入规格

### 2a. 日线 OHLCV

| 字段     | 类型                  | 说明     |
| -------- | --------------------- | -------- |
| `date`   | `string (YYYY-MM-DD)` | 交易日期 |
| `open`   | `float`               | 开盘价   |
| `high`   | `float`               | 最高价   |
| `low`    | `float`               | 最低价   |
| `close`  | `float`               | 收盘价   |
| `volume` | `int`                 | 成交量   |

- **最小数据量：** 252 个交易日（1 年）。其中前 200 个交易日用于 SMA(200) 的初始化，至少保留 52 个交易日的有效输出。
- **推荐数据量：** 300 个交易日，以确保所有均线完成初始化后仍有 ≥100 个交易日的有效信号。

### 2b. 周线 OHLCV

字段与日线相同。

- **最小数据量：** 52 周（1 年）。其中前 40 周用于 SMA(40) 初始化，至少保留 12 周的有效输出。
- **推荐数据量：** 80 周。

### 2c. 缺失数据处理

| 场景                 | 处理规则                                                   |
| -------------------- | ---------------------------------------------------------- |
| 单日缺失（节假日等） | 跳过该日，不插值；均线窗口仅统计实际交易日                 |
| 连续缺失 ≤5 个交易日 | 正常跳过，均线计算不受影响                                 |
| 连续缺失 >5 个交易日 | 标记 `data_gap_warning = true`，均线计算继续但输出附带警告 |
| 数据量不足初始化期   | 该均线字段输出 `null`，不参与排列/趋势判定                 |
| `volume = 0` 或缺失  | 本模块不使用成交量，不影响计算                             |

## 3. 均线计算

### 3a. SMA（简单移动平均线）

$$\text{SMA}(N)_t = \frac{1}{N} \sum_{i=0}^{N-1} C_{t-i}$$

其中 $C_t$ 为第 $t$ 日的收盘价。

| 均线     | 周期 N | 时间框架 | 初始化期              | 用途         |
| -------- | ------ | -------- | --------------------- | ------------ |
| SMA(20)  | 20     | 日线     | 前 20 个交易日无输出  | 短期趋势     |
| SMA(50)  | 50     | 日线     | 前 50 个交易日无输出  | 中期趋势     |
| SMA(200) | 200    | 日线     | 前 200 个交易日无输出 | 长期趋势     |
| SMA(10)  | 10     | 周线     | 前 10 周无输出        | 周线短期趋势 |
| SMA(40)  | 40     | 周线     | 前 40 周无输出        | 周线长期趋势 |

### 3b. EMA（指数移动平均线）

$$\text{EMA}(N)_t = C_t \times k + \text{EMA}(N)_{t-1} \times (1 - k)$$

其中 $k = \frac{2}{N + 1}$（平滑因子）。

**初始化：** 第一个 EMA 值等于前 $N$ 日收盘价的 SMA：

$$\text{EMA}(N)_N = \text{SMA}(N)_N$$

从第 $N+1$ 日起使用递推公式。

| 均线    | 周期 N | 平滑因子 k | 时间框架 | 用途           |
| ------- | ------ | ---------- | -------- | -------------- |
| EMA(10) | 10     | 0.1818     | 日线     | 短期动量方向   |
| EMA(21) | 21     | 0.0952     | 日线     | 中短期动量方向 |

### 3c. 均线斜率

均线斜率用于趋势判定，定义为最近 $W$ 个周期的线性回归斜率，标准化为百分比：

$$\text{slope}(MA, W)_t = \frac{\text{linreg\_slope}(MA_{t-W+1}, \ldots, MA_t)}{MA_t} \times 100$$

| 均线         | 回溯窗口 W | 用途         |
| ------------ | ---------- | ------------ |
| SMA(20)      | 10         | 日线短期斜率 |
| SMA(50)      | 20         | 日线中期斜率 |
| SMA(200)     | 50         | 日线长期斜率 |
| 周线 SMA(10) | 5          | 周线短期斜率 |
| 周线 SMA(40) | 10         | 周线长期斜率 |

**斜率方向判定阈值：**

- `上升`：slope > +0.05%
- `平坦`：-0.05% ≤ slope ≤ +0.05%
- `下降`：slope < -0.05%

## 4. 均线排列判定

均线排列描述当前价格与日线五条均线的相对位置关系。判定基于最新收盘价 $C$ 与各均线值的比较。

### 4a. 六个条件

| 编号 | 条件                               | 含义                     |
| ---- | ---------------------------------- | ------------------------ |
| C1   | $C > \text{EMA}(10)$               | 价格在最短期均线上方     |
| C2   | $C > \text{EMA}(21)$               | 价格在中短期均线上方     |
| C3   | $C > \text{SMA}(20)$               | 价格在短期均线上方       |
| C4   | $C > \text{SMA}(50)$               | 价格在中期均线上方       |
| C5   | $C > \text{SMA}(200)$              | 价格在长期均线上方       |
| C6   | $\text{SMA}(50) > \text{SMA}(200)$ | 金叉结构（中期强于长期） |

### 4b. 排列判定规则

| 排列状态            | 判定条件                          | 含义                                 |
| ------------------- | --------------------------------- | ------------------------------------ |
| `fully_bullish`     | C1 ∧ C2 ∧ C3 ∧ C4 ∧ C5 ∧ C6       | 价格在所有均线上方，且均线呈多头排列 |
| `partially_bullish` | (C4 ∧ C5 ∧ C6) ∧ ¬(C1 ∧ C2 ∧ C3)  | 中长期多头结构完好，短期有回调       |
| `fully_bearish`     | ¬C1 ∧ ¬C2 ∧ ¬C3 ∧ ¬C4 ∧ ¬C5 ∧ ¬C6 | 价格在所有均线下方，且均线呈空头排列 |
| `mixed`             | 以上均不满足                      | 多空交织，无明确方向                 |

**判定优先级：** `fully_bullish` → `fully_bearish` → `partially_bullish` → `mixed`。按此顺序逐条检查，第一个满足即为结果。

## 5. 趋势判定规则

### 5a. 日线趋势 (`trend_daily`)

日线趋势由三个维度综合判定：均线排列、均线斜率、价格与关键均线的位置。

**看多条件（满足 ≥3 条 → `bullish`）：**

| 编号 | 条件                                                    | 说明                 |
| ---- | ------------------------------------------------------- | -------------------- |
| D1   | `ma_alignment` ∈ {`fully_bullish`, `partially_bullish`} | 均线排列偏多         |
| D2   | SMA(50) 斜率 `上升`                                     | 中期均线方向向上     |
| D3   | SMA(200) 斜率 `上升` 或 `平坦`                          | 长期均线未转空       |
| D4   | $C > \text{SMA}(50)$                                    | 价格站稳中期均线上方 |
| D5   | EMA(10) > EMA(21)                                       | 短期均线金叉         |

**看空条件（满足 ≥3 条 → `bearish`）：**

| 编号 | 条件                                               | 说明                           |
| ---- | -------------------------------------------------- | ------------------------------ |
| B1   | `ma_alignment` ∈ {`fully_bearish`, `mixed`} 且 ¬C5 | 排列偏空且价格在 SMA(200) 下方 |
| B2   | SMA(50) 斜率 `下降`                                | 中期均线方向向下               |
| B3   | SMA(200) 斜率 `下降`                               | 长期均线方向向下               |
| B4   | $C < \text{SMA}(50)$                               | 价格在中期均线下方             |
| B5   | EMA(10) < EMA(21)                                  | 短期均线死叉                   |

**中性：** 既不满足看多（<3 条）也不满足看空（<3 条）→ `neutral`。

### 5b. 周线趋势 (`trend_weekly`)

周线趋势基于周线均线和价格位置判定。

**看多条件（满足 ≥2 条 → `bullish`）：**

| 编号 | 条件                                  | 说明                     |
| ---- | ------------------------------------- | ------------------------ |
| W1   | $C_w > \text{SMA}_w(10)$              | 周线收盘价在短期均线上方 |
| W2   | $C_w > \text{SMA}_w(40)$              | 周线收盘价在长期均线上方 |
| W3   | $\text{SMA}_w(10) > \text{SMA}_w(40)$ | 周线均线金叉             |
| W4   | SMA_w(10) 斜率 `上升`                 | 周线短期均线方向向上     |

**看空条件（满足 ≥2 条 → `bearish`）：**

| 编号 | 条件                                  | 说明                     |
| ---- | ------------------------------------- | ------------------------ |
| X1   | $C_w < \text{SMA}_w(10)$              | 周线收盘价在短期均线下方 |
| X2   | $C_w < \text{SMA}_w(40)$              | 周线收盘价在长期均线下方 |
| X3   | $\text{SMA}_w(10) < \text{SMA}_w(40)$ | 周线均线死叉             |
| X4   | SMA_w(10) 斜率 `下降`                 | 周线短期均线方向向下     |

**中性：** 既不满足看多也不满足看空 → `neutral`。

## 6. 支撑/阻力识别算法

### 6a. 波段高低点定义

使用局部极值法识别波段高低点：

- **波段高点（Swing High）：** 第 $t$ 日的最高价 $H_t$ 满足：$H_t > H_{t-i}$ 且 $H_t > H_{t+i}$，对所有 $1 \leq i \leq N$。
- **波段低点（Swing Low）：** 第 $t$ 日的最低价 $L_t$ 满足：$L_t < L_{t-i}$ 且 $L_t < L_{t+i}$，对所有 $1 \leq i \leq N$。

| 参数      | 值  | 说明                      |
| --------- | --- | ------------------------- |
| N（日线） | 5   | 前后各 5 日，即 11 日窗口 |
| N（周线） | 3   | 前后各 3 周，即 7 周窗口  |

> 使用该定义时，最近 $N$ 个交易日/周的数据无法产生波段点（需要右侧确认）。

### 6b. 扫描逻辑

1. **回溯范围：** 最近 52 周（约 252 个交易日）的日线数据 + 52 周的周线数据。
2. **收集候选价位：**
   - 日线波段高点的 $H_t$ → 候选阻力位
   - 日线波段低点的 $L_t$ → 候选支撑位
   - 周线波段高点的 $H_t$ → 候选阻力位（权重更高）
   - 周线波段低点的 $L_t$ → 候选支撑位（权重更高）
3. **当前价格分界：** 以最新收盘价 $C$ 为界，$\text{候选价位} < C$ 归入支撑候选，$\text{候选价位} > C$ 归入阻力候选。波段低点高于 $C$ 可作为阻力，波段高点低于 $C$ 可作为支撑。

### 6c. 价位聚类合并

相近的候选价位合并为一个支撑/阻力区域，避免输出过多冗余价位。

**合并算法：**

1. 将所有候选价位按价格升序排列。
2. 从最低价位开始，以当前价位为簇中心，所有与该中心价格差异 ≤ 1.5% 的价位归入同一簇。
3. 簇的代表价位 = 簇内所有价位的加权平均值：
   - 周线波段点权重 = 2
   - 日线波段点权重 = 1
   - 被触碰次数 > 2 的价位额外 ×1.5
4. 开始下一个未被归入任何簇的价位，重复步骤 2-3。

### 6d. 筛选与排序

从合并后的价位中，按以下规则筛选最终输出：

| 规则                             | 说明                                                                            |
| -------------------------------- | ------------------------------------------------------------------------------- |
| 最多保留支撑位 3 个、阻力位 3 个 | 避免信息过载                                                                    |
| 优先级排序                       | ① 簇内包含周线波段点 > 仅日线波段点；② 被触碰次数多 > 少；③ 距离当前价格近 > 远 |
| 距离过滤                         | 排除距当前价格 > 30% 的价位（过远的支撑/阻力实用性低）                          |

## 7. 日线/周线共振规则

共振规则是本模块的核心输出逻辑，决定结构信号的可信度。

### 7a. 方向一致（共振增强）

| 组合                                               | 效果                           |
| -------------------------------------------------- | ------------------------------ |
| `trend_daily = bullish` ∧ `trend_weekly = bullish` | 结构信号 = `bullish`，高可信度 |
| `trend_daily = bearish` ∧ `trend_weekly = bearish` | 结构信号 = `bearish`，高可信度 |

### 7b. 方向冲突（降级处理）

| 组合                                               | 效果                               | 说明                               |
| -------------------------------------------------- | ---------------------------------- | ---------------------------------- |
| `trend_daily = bullish` ∧ `trend_weekly = bearish` | 结构信号 = `neutral`               | 短期反弹可能，但中期趋势不支持     |
| `trend_daily = bearish` ∧ `trend_weekly = bullish` | 结构信号 = `neutral`               | 短期回调，但中期趋势仍完好         |
| `trend_daily = bullish` ∧ `trend_weekly = neutral` | 结构信号 = `bullish`（降低可信度） | 日线主导，但缺乏周线确认           |
| `trend_daily = bearish` ∧ `trend_weekly = neutral` | 结构信号 = `bearish`（降低可信度） | 日线主导，但缺乏周线确认           |
| `trend_daily = neutral` ∧ `trend_weekly = bullish` | 结构信号 = `neutral`               | 周线多头但日线无方向，等待日线确认 |
| `trend_daily = neutral` ∧ `trend_weekly = bearish` | 结构信号 = `neutral`               | 周线空头但日线无方向，等待日线确认 |
| `trend_daily = neutral` ∧ `trend_weekly = neutral` | 结构信号 = `neutral`               | 完全无方向                         |

### 7c. 边界情况

| 场景                                          | 处理规则                                                                                          |
| --------------------------------------------- | ------------------------------------------------------------------------------------------------- |
| 周线数据不足（SMA(40) 未初始化）              | 仅使用日线趋势，`trend_weekly = null`，共振规则不适用，输出附带 `weekly_data_insufficient = true` |
| 趋势刚刚转向（如 SMA(50) 金叉发生在 ≤3 日内） | 正常输出，但不额外增强信号；由下游模块的突破确认决定是否 `actionable`                             |
| 价格恰好在某均线上（差异 < 0.1%）             | 视为"在均线上方"，即满足条件                                                                      |

## 8. 输出字段

本模块输出以下字段，供聚合器和下游模块消费：

| 字段             | 类型      | 取值范围                                                       | 说明                              |
| ---------------- | --------- | -------------------------------------------------------------- | --------------------------------- |
| `trend_daily`    | `string`  | `bullish \| bearish \| neutral`                                | 日线趋势方向                      |
| `trend_weekly`   | `string`  | `bullish \| bearish \| neutral \| null`                        | 周线趋势方向；数据不足时为 `null` |
| `ma_alignment`   | `string`  | `fully_bullish \| partially_bullish \| mixed \| fully_bearish` | 均线排列状态                      |
| `key_support`    | `[float]` | 最多 3 个，降序排列                                            | 关键支撑位（离当前价格从近到远）  |
| `key_resistance` | `[float]` | 最多 3 个，升序排列                                            | 关键阻力位（离当前价格从近到远）  |

### 聚合器消费规则

聚合器基于本模块的输出生成结构信号：

- `bullish`：`trend_daily = bullish` 且 `trend_weekly = bullish`，且 `ma_alignment` ∈ {`fully_bullish`, `partially_bullish`}
- `bearish`：`trend_daily = bearish` 且 `trend_weekly = bearish`，且价格/均线结构明确走弱（`ma_alignment` ∈ {`fully_bearish`, `mixed`} 且 $C < \text{SMA}(200)$）
- `neutral`：多空周期冲突或仅单周期成立

结构信号权重为 0.35，在模块评分中占比最高。

## 9. 示例

### 示例 A：多头排列（共振增强）

**背景：** 某股票当前收盘价 $C = 155.00$。

**日线均线数据：**

| 均线     | 值     | 斜率               | 价格位置                |
| -------- | ------ | ------------------ | ----------------------- |
| EMA(10)  | 153.20 | —                  | $C > \text{EMA}(10)$ ✓  |
| EMA(21)  | 151.80 | —                  | $C > \text{EMA}(21)$ ✓  |
| SMA(20)  | 152.00 | +0.12%（上升）     | $C > \text{SMA}(20)$ ✓  |
| SMA(50)  | 148.50 | +0.08%（上升）     | $C > \text{SMA}(50)$ ✓  |
| SMA(200) | 135.00 | +0.03%（平坦偏上） | $C > \text{SMA}(200)$ ✓ |

**均线排列判定：**

- C1–C5 全部满足，C6: SMA(50)=148.50 > SMA(200)=135.00 ✓
- → `ma_alignment = fully_bullish`

**日线趋势判定：**

- D1: `fully_bullish` ✓ D2: SMA(50) 上升 ✓ D3: SMA(200) 平坦偏上 ✓ D4: C > SMA(50) ✓ D5: EMA(10) > EMA(21) ✓
- 5/5 条件满足（≥3）→ `trend_daily = bullish`

**周线数据：**

| 均线      | 值     | 斜率           |
| --------- | ------ | -------------- |
| SMA_w(10) | 150.50 | +0.10%（上升） |
| SMA_w(40) | 140.20 | +0.06%（上升） |

周线收盘价 $C_w = 155.00$。

**周线趋势判定：**

- W1: C_w > SMA_w(10) ✓ W2: C_w > SMA_w(40) ✓ W3: SMA_w(10) > SMA_w(40) ✓ W4: SMA_w(10) 上升 ✓
- 4/4 条件满足（≥2）→ `trend_weekly = bullish`

**共振判定：**

- 日线 bullish + 周线 bullish → 结构信号 = `bullish`，高可信度

**支撑/阻力：**

- 波段低点识别到：$148.00$（日线）、$140.00$（周线）、$135.50$（日线）
- 波段高点识别到：$158.00$（周线）、$162.50$（日线）

**最终输出：**

```json
{
  "trend_daily": "bullish",
  "trend_weekly": "bullish",
  "ma_alignment": "fully_bullish",
  "key_support": [148.0, 140.0, 135.5],
  "key_resistance": [158.0, 162.5]
}
```

聚合器结构信号 → `bullish`（权重 0.35 × +1 = +0.35）。

### 示例 B：多空冲突（降级处理）

**背景：** 某股票当前收盘价 $C = 142.00$。

**日线均线数据：**

| 均线     | 值     | 斜率           | 价格位置                |
| -------- | ------ | -------------- | ----------------------- |
