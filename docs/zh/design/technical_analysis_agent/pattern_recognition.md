# 形态识别子模块

## 1. 模块职责与边界

### 职责

形态识别子模块（④）是技术分析 Agent 的信号生成层，负责：

1. 基于上游子 Agent 输出的趋势、动量、价量数据，扫描当前价格结构是否匹配预定义的 8 种经典形态
2. 对匹配的形态进行质量评分（`high / medium / low`）
3. 计算目标价、止损价和风险收益比
4. 生成可供聚合器直接消费的结构化输出

### 边界

| 范围内 | 范围外 |
|---|---|
| 8 种预定义的延续/反转形态识别 | 日内形态（如 5 分钟级别旗形） |
| 基于几何结构 + 量能 + 趋势背景的规则匹配 | 基于机器学习的形态分类 |
| 目标价与止损价的量化计算 | 仓位大小计算（由风险管理 Agent 负责） |
| 形态质量评分 | 最终交易决策（由人类决策者负责） |

### 依赖关系

本模块**不直接消费原始 OHLCV 数据**，而是依赖 ①②③ 的输出字段。运行顺序上，④ 必须在 ①②③ 全部完成后才启动。

```
① 多周期结构分析 ──┐
② 动量与强度量化 ──┼──▶ ④ 形态识别
③ 价量关系分析 ────┘
```

## 2. 输入规格

### 2a. 来自 ① 多周期结构分析的字段

| 字段 | 类型 | 用途 |
|---|---|---|
| `trend_daily` | `bullish \| bearish \| neutral` | 判断形态前提条件中的趋势方向 |
| `trend_weekly` | `bullish \| bearish \| neutral` | 中期趋势确认，提升形态可信度 |
| `ma_alignment` | `fully_bullish \| partially_bullish \| mixed \| fully_bearish` | 形态质量评分的趋势维度 |
| `key_support` | `[float]` | 止损位校验、平台/箱体形态的支撑位识别 |
| `key_resistance` | `[float]` | 突破目标位校验、压力线识别 |

### 2b. 来自 ② 动量与强度量化的字段

| 字段 | 类型 | 用途 |
|---|---|---|
| `rsi` | `float` | 动量背景判断；超买/超卖区间影响形态可信度 |
| `rsi_signal` | `overbought \| healthy \| oversold` | 形态质量评分的动量维度 |
| `macd_signal` | `bullish_cross \| bearish_cross \| flat` | 触发确认的辅助条件 |
| `adx` | `float` | 趋势强度；ADX < 20 的震荡市中形态可信度降级 |
| `relative_strength` | `float` | 多头形态要求 RS > 1.0；空头形态要求 RS < 1.0 |

### 2c. 来自 ③ 价量关系分析的字段

| 字段 | 类型 | 用途 |
|---|---|---|
| `obv_trend` | `rising \| falling \| flat` | 量能趋势确认 |
| `obv_divergence` | `bullish \| bearish \| none` | 背离信号影响形态质量 |
| `breakout_confirmed` | `boolean` | 突破形态的量能确认 |
| `breakdown_confirmed` | `boolean` | 破位形态的量能确认 |
| `volume_pattern` | `accumulation \| distribution \| neutral \| pullback_healthy \| bounce_weak` | 形态期间的量能特征判断 |

### 2d. 原始 OHLCV 数据（直接访问）

形态识别需要直接读取原始 OHLCV 数据用于几何结构计算：

| 字段 | 说明 |
|---|---|
| 日线 OHLCV | 近 60 个交易日的完整数据，用于形态几何结构的精确计算 |
| 20 日均量 | 成交量基准线 |
| ATR(14) | 来自 ⑤ 风险指标计算，用于止损距离 |

## 3. 形态识别规格

### 通用定义

以下变量在各形态规格中反复使用：

| 变量 | 定义 |
|---|---|
| `ATR` | 14 周期 Average True Range |
| `avg_vol_20` | 近 20 个交易日的日均成交量 |
| `pivot_high` | 局部波段高点（左右各 ≥3 根 K 线不高于该点） |
| `pivot_low` | 局部波段低点（左右各 ≥3 根 K 线不低于该点） |
| `flagpole` | 旗杆——形态前的一段急涨/急跌走势 |
| `breakout_price` | 突破枢轴价（形态上沿或关键阻力位） |
| `breakdown_price` | 破位枢轴价（形态下沿或关键支撑位） |

### 3.1 VCP（Volatility Contraction Pattern，波动率收缩形态）

**理论基础：** Mark Minervini 提出的经典多头延续形态。核心逻辑是股价经历多轮振幅递减的回调，每轮回调淘汰一批浮筹，当供给枯竭、波动率压缩到极致时，少量买盘即可推动突破。

#### 前提条件

1. `trend_daily = bullish`，且 `ma_alignment ∈ {fully_bullish, partially_bullish}`
2. 形态起始前至少有一段 ≥20% 的上升趋势（旗杆），或价格位于 52 周高点附近 25% 范围内
3. `relative_strength ≥ 1.0`

#### 结构约束

| 约束 | 规格 |
|---|---|
| 持续周期 | 10–40 个交易日 |
| 收缩次数 | ≥2 次回调（T1、T2、...），推荐 3 次 |
| 回调幅度递减 | 每次回调幅度 < 前次的 60%–80%；例如 T1 = 15%、T2 = 8%、T3 = 4% |
| 最大首次回调 | T1 ≤ 30%（超过则判定为深度回调而非 VCP） |
| 最后一次回调 | 末次回调幅度 ≤ 10%，最理想 ≤ 5% |
| 枢轴线 | 每次反弹的高点应大致持平（形成水平阻力），允许 ±2% 偏差 |

#### 量能要求

| 阶段 | 要求 |
|---|---|
| 每次回调 | 成交量逐次递减；T(n) 阶段日均量 < T(n-1) 阶段日均量 |
| 末次收缩 | 末次收缩区间的日均量 < 0.7 × `avg_vol_20` |
| 突破日 | 成交量 ≥ 1.5 × `avg_vol_20` |

#### 触发条件

- **价格：** 收盘价突破枢轴线（各次反弹高点连线），即 `close > pivot_high`
- **量能：** 突破当日成交量 ≥ 1.5 × `avg_vol_20`
- **确认：** 以收盘价为准（盘中突破不算确认）

#### 目标价计算

```
target_price = breakout_price + flagpole_height × projection_factor
```

- `flagpole_height` = VCP 起始前那段上涨的高度（旗杆高点 - 旗杆起涨点）
- `projection_factor` = 0.50–1.00（默认 0.75）
- 若目标价位恰好在 `key_resistance` 附近（±2%），则下调至该阻力位

#### 止损价计算

```
stop_loss_price = last_pivot_low - 1.0 × ATR
```

- `last_pivot_low` = VCP 最后一次收缩的最低点

#### 失效条件

1. 任一收缩的回调幅度 > 前次回调幅度（振幅扩张而非收缩）
2. 突破后 3 个交易日内收盘价回落至枢轴线下方（假突破）
3. 突破日成交量 < 1.0 × `avg_vol_20`（量能不足）
4. 收盘价跌破 `stop_loss_price`

### 3.2 牛市旗形（Bull Flag）

**理论基础：** 急涨后的短暂休整，旗杆代表强劲买盘，旗面的低量平行回调代表获利盘的有序消化，突破后大概率延续旗杆方向。

#### 前提条件

1. `trend_daily = bullish`
2. 旗杆阶段：在 1–8 个交易日内价格上涨 ≥10%（急涨），且旗杆期间日均量 ≥ 1.5 × `avg_vol_20`
3. `relative_strength ≥ 0.8`

#### 结构约束

| 约束 | 规格 |
|---|---|
| 旗杆长度 | 1–8 个交易日 |
| 旗面持续周期 | 5–15 个交易日 |
| 旗面形状 | 平行下降通道或矩形横盘（上下轨近似平行） |
| 旗面回调幅度 | ≤ 旗杆涨幅的 38.2%（斐波那契回撤） |
| 理想回调幅度 | 旗杆涨幅的 20%–33% |
| 旗面倾斜方向 | 略向下倾斜或水平，不可向上倾斜 |

#### 量能要求

| 阶段 | 要求 |
|---|---|
| 旗杆 | 日均量 ≥ 1.5 × `avg_vol_20` |
| 旗面 | 日均量 ≤ 0.8 × `avg_vol_20`；量能逐步萎缩 |
| 突破日 | 成交量 ≥ 1.5 × `avg_vol_20` |

#### 触发条件

- **价格：** 收盘价突破旗面上轨（下降通道上边界），即 `close > flag_upper_trendline`
- **量能：** 突破当日成交量 ≥ 1.5 × `avg_vol_20`
- **确认：** 收盘价确认

#### 目标价计算

```
target_price = breakout_price + flagpole_height
```

- `flagpole_height` = 旗杆最高点 - 旗杆最低点（即旗杆涨幅的等幅投影）

#### 止损价计算

```
stop_loss_price = flag_low - 1.0 × ATR
```

- `flag_low` = 旗面期间的最低价

#### 失效条件

1. 旗面回调幅度超过旗杆涨幅的 50%（回调过深，买盘不足）
2. 旗面持续超过 20 个交易日（整理时间过长，动能衰竭）
3. 旗面期间出现单日放量下跌（日量 > 1.5 × `avg_vol_20` 且收阴）
4. 收盘价跌破 `stop_loss_price`

### 3.3 Flat Base（平台整理突破）

**理论基础：** 来自 William O'Neil 的 CAN SLIM 体系。强势股在大涨后横盘整理，窄幅振荡消化获利盘，一旦突破平台上沿，往往开启新一轮上涨。

#### 前提条件

1. `trend_daily = bullish`，且 `trend_weekly = bullish`
2. 平台起始前至少有一段 ≥20% 的上升趋势
3. `relative_strength ≥ 1.0`
4. `ma_alignment ∈ {fully_bullish, partially_bullish}`

#### 结构约束

| 约束 | 规格 |
|---|---|
| 持续周期 | 15–60 个交易日 |
| 价格振幅 | 整理期间最高价与最低价之差 ≤ 15%（以最高价为基准） |
| 理想振幅 | ≤ 10% |
| 平台上沿 | 整理期间的最高收盘价（或多次触及的阻力价位） |
| 平台下沿 | 整理期间的最低收盘价 |
| 价格重心 | 收盘价中位数应位于平台上半部分（买方支撑明显） |

#### 量能要求

| 阶段 | 要求 |
|---|---|
| 整理期间 | 日均量 ≤ 0.7 × `avg_vol_20`（量能持续低迷） |
| 量能趋势 | `obv_trend = rising` 或 `flat`（不可为 falling） |
| 突破日 | 成交量 ≥ 1.5 × `avg_vol_20` |

#### 触发条件

- **价格：** 收盘价突破平台上沿，即 `close > platform_high`
- **量能：** 突破当日成交量 ≥ 1.5 × `avg_vol_20`
- **确认：** 收盘价确认

#### 目标价计算

```
target_price = breakout_price + platform_height
```

- `platform_height` = 平台上沿 - 平台下沿

#### 止损价计算

```
stop_loss_price = platform_low - 1.0 × ATR
```

- `platform_low` = 平台期间的最低价

#### 失效条件

1. 整理期间价格振幅 > 20%（不再是"平台"，而是宽幅震荡）
2. 整理期间 `obv_trend = falling`（暗中派发）
3. 整理超过 65 个交易日（时间过长，关注度消散）
4. 突破后 3 个交易日内收盘价回落至平台上沿下方（假突破）
5. 收盘价跌破 `stop_loss_price`

### 3.4 上升三角形（Ascending Triangle）

**理论基础：** 经典的多头蓄势形态。水平阻力线代表固定的卖压价位，而逐步抬高的低点说明买方越来越急切，愿意以更高价格接货，最终买压突破卖压。

#### 前提条件

1. `trend_daily ∈ {bullish, neutral}`（上升三角形可出现在上升趋势中作为延续形态，也可出现在横盘中作为突破形态）
2. 至少有 2 个 `pivot_high` 构成的水平阻力线
3. 至少有 2 个 `pivot_low` 构成上升趋势线（每个低点高于前一个）

#### 结构约束

| 约束 | 规格 |
|---|---|
| 持续周期 | 15–60 个交易日 |
| 水平阻力线 | ≥2 次触及同一价位（±1% 容差），触及次数越多越好 |
| 上升支撑线 | ≥2 个依次抬高的低点，低点间隔 ≥5 个交易日 |
| 三角形收敛 | 阻力线与支撑线趋于收敛（预期交汇） |
| 最宽处高度 | 三角形第一个高点与第一个低点的价差 |
| 突破位置 | 理想在三角形 50%–75% 处突破（距离起始点的时间比例） |

#### 量能要求

| 阶段 | 要求 |
|---|---|
| 形态形成期间 | 成交量总体递减（随着价格收敛） |
| 每次触及阻力线 | 量能逐次递减（卖压减弱） |
| 每次触及支撑线反弹 | 反弹日量能 > 回落日量能（买方主导） |
| 突破日 | 成交量 ≥ 1.5 × `avg_vol_20` |

#### 触发条件

- **价格：** 收盘价突破水平阻力线，即 `close > resistance_line`
- **量能：** 突破当日成交量 ≥ 1.5 × `avg_vol_20`
- **确认：** 收盘价确认；理想情况下突破后回踩阻力线（变支撑）不破

#### 目标价计算

```
target_price = breakout_price + triangle_height
```

- `triangle_height` = 三角形最宽处的高度（第一个高点 - 第一个低点）

#### 止损价计算

```
stop_loss_price = last_support_low - 1.0 × ATR
```

- `last_support_low` = 上升支撑线上最后一个低点

#### 失效条件

1. 价格向下跌破上升支撑线（多头逻辑瓦解）
2. 突破发生在三角形 >85% 位置（动能不足）
3. 阻力线被触及超过 5 次仍未突破（过度消耗）
4. 收盘价跌破 `stop_loss_price`

### 3.5 杯柄形态（Cup and Handle）

**理论基础：** William O'Neil 的经典形态。圆弧底（杯体）代表一轮从恐慌到企稳再到信心恢复的洗盘过程，柄部是最后一批浮筹的清洗，突破柄部高点后开启新升浪。

#### 前提条件

1. `trend_weekly = bullish`（中长期趋势向上）
2. 杯体起始前股价位于或接近阶段性高点（杯体左沿代表前高）
3. `relative_strength ≥ 0.8`

#### 结构约束

| 约束 | 规格 |
|---|---|
| 杯体持续周期 | 30–50 个交易日（本模块为中短期压缩版） |
| 杯体形状 | 圆弧底（U 形），不可为 V 形急跌急涨 |
| 杯体深度 | 杯体左沿到杯底的回撤 ≤ 35%；理想范围 15%–30% |
| 杯体对称性 | 右沿价格应达到左沿的 90%–100%（允许右沿略低于左沿） |
| 柄部持续周期 | 5–15 个交易日 |
| 柄部回调深度 | ≤ 杯体深度的 50%，且绝对幅度 ≤ 15% |
| 柄部位置 | 柄部应形成在杯体上半部分（即柄底 > 杯底） |
| 柄部形状 | 略向下倾斜的旗形或楔形，不可向上倾斜 |

**圆弧底判定方法：** 将杯体周期等分为 3 段（左段、底段、右段），左段收盘价均值 > 底段收盘价均值，右段收盘价均值 > 底段收盘价均值，且右段均值 ≥ 左段均值 × 0.90。

#### 量能要求

| 阶段 | 要求 |
|---|---|
| 杯体左侧下跌段 | 成交量偏高（恐慌抛售） |
| 杯体底部 | 成交量萎缩（抛压枯竭） |
| 杯体右侧上升段 | 成交量逐步回升（信心恢复） |
| 柄部 | 成交量显著萎缩，日均量 ≤ 0.7 × `avg_vol_20` |
| 突破日 | 成交量 ≥ 1.5 × `avg_vol_20` |

#### 触发条件

- **价格：** 收盘价突破柄部高点（即杯体右沿价位），即 `close > handle_high`
- **量能：** 突破当日成交量 ≥ 1.5 × `avg_vol_20`
- **确认：** 收盘价确认

#### 目标价计算

```
target_price = breakout_price + cup_depth
```

- `cup_depth` = 杯体左沿价格 - 杯体最低价

#### 止损价计算

```
stop_loss_price = handle_low - 1.0 × ATR
```

- `handle_low` = 柄部最低价

#### 失效条件

1. 杯体为 V 形（急跌急涨，缺乏充分洗盘）
2. 杯体深度 > 40%（回撤过深，趋势可能已破坏）
3. 柄部回调深度 > 杯体深度的 50%（最后浮筹清洗过度，买方疲软）
4. 柄部日均量 > 1.0 × `avg_vol_20`（量能未收缩，浮筹未出清）
5. 收盘价跌破 `stop_loss_price`

### 3.6 熊市旗形（Bear Flag）

**理论基础：** 牛市旗形的镜像。急跌后的低量反弹只是下跌途中的喘息，卖方仍然主导，反弹结束后继续下行。

#### 前提条件

1. `trend_daily = bearish`
2. 旗杆阶段：在 1–8 个交易日内价格下跌 ≥10%（急跌），且旗杆期间日均量 ≥ 1.5 × `avg_vol_20`
3. `relative_strength ≤ 1.0`

#### 结构约束

| 约束 | 规格 |
|---|---|
| 旗杆长度 | 1–8 个交易日 |
| 旗面持续周期 | 5–15 个交易日 |
| 旗面形状 | 平行上升通道或矩形横盘 |
| 旗面反弹幅度 | ≤ 旗杆跌幅的 38.2% |
| 理想反弹幅度 | 旗杆跌幅的 20%–33% |
| 旗面倾斜方向 | 略向上倾斜或水平，不可向下倾斜（那就是直接延续下跌） |

#### 量能要求

| 阶段 | 要求 |
|---|---|
| 旗杆 | 日均量 ≥ 1.5 × `avg_vol_20`（恐慌抛售） |
| 旗面 | 日均量 ≤ 0.8 × `avg_vol_20`（反弹乏力） |
| 破位日 | 成交量 ≥ 1.5 × `avg_vol_20` |

#### 触发条件

- **价格：** 收盘价跌破旗面下轨，即 `close < flag_lower_trendline`
- **量能：** 破位当日成交量 ≥ 1.5 × `avg_vol_20`
- **确认：** 收盘价确认

#### 目标价计算

```
target_price = breakdown_price - flagpole_height
```

- `flagpole_height` = 旗杆最高点 - 旗杆最低点（旗杆跌幅的等幅投影向下）

#### 止损价计算

```
stop_loss_price = flag_high + 1.0 × ATR
```

- `flag_high` = 旗面期间的最高价

#### 失效条件

1. 旗面反弹幅度超过旗杆跌幅的 50%（反弹过强，空头逻辑减弱）
2. 旗面持续超过 20 个交易日
3. 旗面期间出现单日放量上涨（日量 > 1.5 × `avg_vol_20` 且收阳）
4. 收盘价涨破 `stop_loss_price`

---

### 3.7 平台/箱体破位（Breakdown Base）

**理论基础：** 价格在某一区间多次测试支撑位，每次反弹力度递减，说明买方承接力逐步衰竭。当支撑最终被放量击穿，积蓄的卖压集中释放。

#### 前提条件

1. `trend_daily ∈ {bearish, neutral}`
2. `trend_weekly ∈ {bearish, neutral}`（周线不可为明确的 bullish）
3. `volume_pattern ∈ {distribution, neutral}`

#### 结构约束

| 约束 | 规格 |
|---|---|
| 持续周期 | 15–60 个交易日 |
| 支撑线 | ≥2 次触及同一支撑价位（±1% 容差），触及次数越多破位威力越大 |
| 箱体上沿 | 整理期间的最高价（阻力位） |
| 箱体下沿 | 整理期间的最低价（支撑位，即破位枢轴） |
| 反弹力度递减 | 每次反弹的高点依次降低，或反弹幅度逐次缩小 |
| 箱体中间价 | `(箱体上沿 + 箱体下沿) / 2`，用于止损计算 |

#### 量能要求

| 阶段 | 要求 |
|---|---|
| 整理期间 | `obv_trend = falling`（暗中派发），或量能在反弹日递减 |
| 每次触及支撑反弹 | 反弹日量能逐次递减（买方力竭） |
| 破位日 | 成交量 ≥ 1.5 × `avg_vol_20` |

#### 触发条件

- **价格：** 收盘价跌破箱体下沿（支撑线），即 `close < support_line`
- **量能：** 破位当日成交量 ≥ 1.5 × `avg_vol_20`
- **确认：** 收盘价确认

#### 目标价计算

```
target_price = breakdown_price - platform_height
```

- `platform_height` = 箱体上沿 - 箱体下沿

#### 止损价计算

```
stop_loss_price = platform_midpoint + 1.0 × ATR
```

- `platform_midpoint` = `(箱体上沿 + 箱体下沿) / 2`

#### 失效条件

1. 破位后 3 个交易日内收盘价收回支撑线上方（假破位）
2. 破位日成交量 < 1.0 × `avg_vol_20`（量能不足，破位不可信）
3. `obv_trend = rising`（与破位方向矛盾，存在暗中吸筹的可能）
4. 收盘价涨破 `stop_loss_price`

---

### 3.8 下降三角形（Descending Triangle）

**理论基础：** 上升三角形的镜像。水平支撑线代表固定的接盘价位，而逐步压低的高点说明卖方越来越急切，最终卖压突破买方防线。

#### 前提条件

1. `trend_daily ∈ {bearish, neutral}`
2. 至少有 2 个 `pivot_low` 构成水平支撑线
3. 至少有 2 个 `pivot_high` 构成下降趋势线（每个高点低于前一个）

#### 结构约束

| 约束 | 规格 |
|---|---|
| 持续周期 | 15–60 个交易日 |
| 水平支撑线 | ≥2 次触及同一价位（±1% 容差） |
| 下降阻力线 | ≥2 个依次降低的高点，高点间隔 ≥5 个交易日 |
| 三角形收敛 | 支撑线与阻力线趋于收敛 |
| 最宽处高度 | 三角形第一个高点与第一个低点的价差 |
| 破位位置 | 理想在三角形 50%–75% 处破位 |

#### 量能要求

| 阶段 | 要求 |
|---|---|
| 形态形成期间 | 成交量总体递减 |
| 每次触及支撑线反弹 | 反弹量能逐次递减（买方力竭） |
| 每次从高点回落 | 回落日量能 > 反弹日量能（卖方主导） |
| 破位日 | 成交量 ≥ 1.5 × `avg_vol_20` |

#### 触发条件

- **价格：** 收盘价跌破水平支撑线，即 `close < support_line`
- **量能：** 破位当日成交量 ≥ 1.5 × `avg_vol_20`
- **确认：** 收盘价确认

#### 目标价计算

```
target_price = breakdown_price - triangle_height
```

- `triangle_height` = 三角形最宽处的高度（第一个高点 - 第一个低点）

#### 止损价计算

```
stop_loss_price = last_resistance_high + 1.0 × ATR
```

- `last_resistance_high` = 下降阻力线上最后一个高点

#### 失效条件

1. 价格向上突破下降阻力线（空头逻辑瓦解）
2. 破位发生在三角形 >85% 位置（动能不足）
3. 支撑线被触及超过 5 次仍未破位
4. 收盘价涨破 `stop_loss_price`

## 4. 形态质量评分

每个被识别的形态都会获得 `high / medium / low` 的质量评分。评分基于以下五个维度，每个维度打分后取综合结论。

### 评分维度

| 维度 | 权重 | high（2 分） | medium（1 分） | low（0 分） |
|---|---|---|---|---|
| **量能配合** | 30% | 整理期量缩 + 突破/破位放量 ≥1.5× 均量，完全符合规格 | 整理期量缩但突破/破位量能在 1.2×–1.5× 之间 | 整理期量未缩，或突破/破位量能 < 1.2× 均量 |
| **回撤/振幅** | 25% | 回撤幅度在理想范围内（如牛旗 20%–33%、VCP 末次 ≤5%） | 回撤幅度在允许范围内但不在理想范围 | 回撤幅度接近失效阈值 |
| **时间窗口** | 15% | 形态持续时间在标准范围的 50%–80% 区间 | 形态持续时间在允许范围内 | 形态持续时间接近范围边界 |
| **趋势背景** | 20% | `ma_alignment` 完全配合方向，日线周线共振 | 日线方向配合但周线为 neutral | 仅日线方向配合，周线冲突 |
| **动量支持** | 10% | RSI 健康区间 + MACD 信号配合 + RS 优于基准 | 至少两项配合 | 仅一项或零项配合 |

### 综合评分规则

```
quality_score = Σ(维度分数 × 维度权重)
```

- `quality_score ≥ 1.5` → `pattern_quality = "high"`
- `1.0 ≤ quality_score < 1.5` → `pattern_quality = "medium"`
- `quality_score < 1.0` → `pattern_quality = "low"`

### 评分对聚合器的影响

根据 overview 第 250–253 行的聚合规则：

- `high` 或 `medium` 质量的多头形态 → 形态信号 = `bullish`
- `high` 或 `medium` 质量的空头形态 → 形态信号 = `bearish`
- `low` 质量或无有效形态 → 形态信号 = `neutral`

---

## 5. 多形态冲突处理

当扫描器同时检测到多个符合条件的形态时，按以下规则选择最终输出形态：

### 优先级规则

**规则 1：质量优先**

在不同形态之间，优先选择 `pattern_quality` 更高的形态。

```
high > medium > low
```

**规则 2：同质量下，方向与趋势一致优先**

若多个形态质量相同，选择 `pattern_direction` 与 `trend_daily` 方向一致的形态。

**规则 3：同质量同方向下，风险收益比优先**

若多个形态质量、方向均相同，选择 `risk_reward_ratio` 更高的形态。

**规则 4：多空冲突**

若同时检测到多头和空头形态（如上升三角形 + 下降三角形变体），且质量均为 `medium` 或以上：
- 输出 `pattern_direction = "none"`，`pattern_detected = "none"`
- 在 `entry_trigger` 中注明冲突：`"多空形态冲突：[形态A] vs [形态B]，等待方向明确"`

**规则 5：嵌套形态**

若一个大形态内部包含小形态（如 Flat Base 的后段符合 VCP 特征）：
- 输出外层（更大时间跨度）的形态
- 在 `entry_trigger` 描述中附加内层形态信息

### 输出约束

每次扫描仅输出**一个**最终形态（或 `none`），不输出形态列表。多形态选择逻辑在模块内部完成，聚合器只接收最终结论。

---

## 6. 风险收益比计算

`risk_reward_ratio` 衡量潜在收益与潜在风险的比值，是聚合器判定 `setup_state` 的重要输入。

### 计算公式

**多头形态：**

```
risk_reward_ratio = (target_price - breakout_price) / (breakout_price - stop_loss_price)
```

**空头形态：**

```
risk_reward_ratio = (breakdown_price - target_price) / (stop_loss_price - breakdown_price)
```

### 解读

| 范围 | 解读 |
|---|---|
| ≥ 3.0 | 优秀——潜在收益为风险的 3 倍以上 |
| 2.0–3.0 | 良好——标准的可交易水平 |
| 1.5–2.0 | 一般——仅在其他信号强烈配合时考虑 |
| < 1.5 | 不足——不建议入场，形态信号降权 |

### 约束

- 若 `risk_reward_ratio < 1.5`，即使形态结构完美，`pattern_quality` 也强制降为 `low`
- 若 `stop_loss_price` 与 `breakout_price`（或 `breakdown_price`）之差 < 0.5 × ATR，则止损过紧，需调整为 1.0 × ATR 的最小距离

## 7. 入场触发描述生成

`entry_trigger` 是一段人类可读的文本，描述触发入场的具体条件。由模块根据识别到的形态自动生成。

### 生成规则

**模板格式：**

```
"{量能条件}收盘{方向}{枢轴价} {形态名}"
```

**组成部分：**

| 部分 | 生成规则 |
|---|---|
| 量能条件 | 若突破/破位未确认：`"高量"`；若已确认则省略 |
| 方向 | 多头形态：`"突破"`；空头形态：`"跌破"` |
| 枢轴价 | 格式 `"$xxx.xx"`，取 `breakout_price` 或 `breakdown_price` |
| 形态名 | 形态中文名（如 `"VCP 枢轴"`、`"牛旗上轨"`、`"平台上沿"`） |

**生成示例：**

| 形态 | 状态 | entry_trigger |
|---|---|---|
| VCP | 未突破 | `"高量收盘突破 $142.50 VCP 枢轴"` |
| VCP | 已突破确认 | `"已突破 $142.50 VCP 枢轴，持有"` |
| 牛市旗形 | 未突破 | `"高量收盘突破 $88.20 牛旗上轨"` |
| Flat Base | 未突破 | `"高量收盘突破 $215.00 平台上沿"` |
| 上升三角形 | 未突破 | `"高量收盘突破 $67.80 三角形阻力线"` |
| 杯柄形态 | 未突破 | `"高量收盘突破 $310.50 柄部高点"` |
| 熊市旗形 | 未破位 | `"高量收盘跌破 $45.30 熊旗下轨"` |
| 平台箱体破位 | 未破位 | `"高量收盘跌破 $92.00 箱体支撑线"` |
| 下降三角形 | 未破位 | `"高量收盘跌破 $55.60 三角形支撑线"` |
| 无形态 | — | `"无有效形态"` |
| 多空冲突 | — | `"多空形态冲突：[形态A] vs [形态B]，等待方向明确"` |

## 8. 输出字段

本模块的输出与 overview 定义的 schema 保持一致：

| 字段 | 类型 | 说明 |
|---|---|---|
| `pattern_direction` | `bullish \| bearish \| none` | 形态指向的方向；无形态或多空冲突时为 `none` |
| `pattern_detected` | `vcp \| bull_flag \| flat_base \| ascending_triangle \| cup_and_handle \| bear_flag \| breakdown_base \| descending_triangle \| none` | 识别到的形态标识符 |
| `pattern_quality` | `high \| medium \| low` | 形态质量评分 |
| `entry_trigger` | `string` | 入场触发条件的人类可读描述 |
| `target_price` | `float` | 目标价位 |
| `stop_loss_price` | `float` | 止损价位 |
| `risk_reward_ratio` | `float` | 风险收益比 |

### 特殊情况的输出

**未检测到任何形态：**

```json
{
  "pattern_direction": "none",
  "pattern_detected": "none",
  "pattern_quality": "low",
  "entry_trigger": "无有效形态",
  "target_price": null,
  "stop_loss_price": null,
  "risk_reward_ratio": null
}
```

**形态已触发（突破/破位已确认）：**

所有字段正常填充，`entry_trigger` 更新为已确认状态的描述文本。
